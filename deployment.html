Deployment Notes

Stable GitHub Pages workflow.

This page keeps the site update process visible so future changes do not create cache confusion or version-folder sprawl.

Current deployment model

  • Repository: ProfRinsem/sote-command-portal
  • Branch: main
  • Pages folder: / root
  • Stable URL: https://profrinsem.github.io/sote-command-portal/
  • Site version: v0.9 True Baseline Build

Update rule

Do not upload the ZIP itself. Copy the unzipped contents into the existing repo so index.html remains at the root. Keep stable page names and use versioned asset references for cache-busting.

Release Checklist

Before sharing a link

Safety

No sensitive data

Do not publish passwords, internal-only IPs, student data, unpublished operational details, or production access paths.

Cache

Stable paths

Keep the same URL and same page filenames. Change content and asset version numbers, not the public path.

Control

Document the release

Update README, CHANGELOG, footer version, and deployment notes so the site tells us what build is live.
